# Code of Conduct

Sentinel Prime is a public-facing premium cybersecurity repository with dashboard, security-operations, and enterprise product context. All public interaction should remain professional, respectful, security-conscious, and legally calm.

## Expected Conduct

Participants are expected to:

- communicate respectfully and in good faith
- keep discussions focused on public documentation, governance, repository quality, and lawful cybersecurity interface use cases
- avoid publishing sensitive technical, commercial, customer, telemetry, or incident details in public channels
- respect the Public Edition and Private Enterprise Edition boundary
- disclose security concerns through SECURITY.md rather than public issues
- use precise, professional language suitable for CISOs, security leaders, procurement teams, and enterprise evaluators

## Unacceptable Conduct

The following behavior is not acceptable:

- harassment, personal attacks, discriminatory language, or intimidation
- requests for offensive operations, unauthorized access, credential misuse, data misuse, evasion, persistence, or bypass guidance
- publication of secrets, API keys, tokens, detection rules, telemetry data, customer data, incident records, internal infrastructure references, or private implementation details
- attempts to extract dashboard logic, SOC workflows, threat-intelligence pipelines, detection logic, private prompts, analyst playbooks, restricted research, or commercial implementation details
- spam, impersonation, misleading claims, or inflammatory language
- public vulnerability disclosure before coordination with the maintainer

## Security-Sensitive Communication

Do not post sensitive reports publicly. Use the process in SECURITY.md for vulnerability reports or repository safety concerns.

If a public issue contains sensitive information, the maintainer may edit, hide, lock, or remove it to protect users, customers, buyers, and the integrity of the project.

## Enforcement

The maintainer may take appropriate action to keep the repository safe and professional, including closing issues, declining pull requests, removing unsafe content, or restricting participation.

Enforcement decisions are made at the maintainer's discretion and are intended to preserve the lawful, enterprise-safe scope of the repository.

## Scope

This Code of Conduct applies to public repository interactions, including issues, pull requests, discussions, comments, and any public communication connected to Sentinel Prime.
