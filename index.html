Everything You Need to Know

Clear answers before you commit. No ambiguity. No enterprise sales-speak.

What exactly is Sentinel Prime? Is it real security software?
+

No — Sentinel Prime is not production security software. It is a Proof-of-Concept (PoC) UI Kit — a fully functional HTML/CSS/JavaScript wireframe and design template that demonstrates what an enterprise cybersecurity dashboard could look like. All data shown (alert counts, IP addresses, threat feeds) is simulated. There is no backend, no real threat detection, and no live data pipeline. It is a design artefact — a premium-grade starting point for developers, agencies, and product teams building actual security products.

What frameworks or libraries does Sentinel Prime require?
+

Almost none. Sentinel Prime is built on pure HTML, CSS3, and vanilla JavaScript — no React, no Vue, no Angular, no Tailwind, no Bootstrap. The only external dependency is Chart.js v4.4.1 (loaded from CDN) for the interactive charts. Every other effect — scanlines, glitch animations, terminal simulation, sidebar navigation — is pure CSS and vanilla JS. You can drop it into any project, any stack, or run it directly from the filesystem with zero build steps.

How many modules are included and what do they cover?
+

Sentinel Prime ships with 8 fully-designed enterprise modules, all navigable from the sidebar: (1) Threat Dashboard — alert trends, severity breakdown, live threat feed; (2) Network Monitor — ingress/egress charts, top talkers, geo-threat simulation; (3) Firewall Analytics — block-reason horizontal bar charts; (4) Vulnerability Management — CVE inventory, patch velocity charts; (5) UEBA — user behaviour anomaly scoring and doughnut chart; (6) Incident Response — SOAR-style queue with MITRE ATT&CK tags; (7) Threat Hunt Terminal — live interactive CLI simulation; (8) Compliance & Reports — NIST CSF radar chart, incident trend lines.

Can I connect real data sources to the dashboard?
+

Yes — that is precisely the intended use case for commercial licences. The chart data in Sentinel Prime is generated via simple JavaScript arrays and Chart.js datasets. Replacing simulated data with real API responses (REST, GraphQL, WebSocket streams from SIEM platforms like Splunk, Elastic SIEM, Microsoft Sentinel, or CrowdStrike) requires only straightforward fetch calls and data-binding in the existing JS structure. The UI architecture is deliberately backend-agnostic. No proprietary data-binding layer stands between you and your data source.

What browsers and devices does Sentinel Prime support?
+

Sentinel Prime is tested and fully functional on all modern browsers: Chrome 120+, Firefox 121+, Safari 17+, Edge 120+. The layout uses CSS Grid and Flexbox for responsive behaviour. The dashboard is primarily designed for desktop/widescreen viewports (1280px and above) — as would be expected from an enterprise SOC interface — but degrades gracefully on tablets. True mobile support is not a design goal for this release, consistent with the target use case (analysts at workstations).
