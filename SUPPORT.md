# Support

This repository is the Public Edition of Sentinel Prime. Support is limited to public documentation, governance, repository hygiene, and safe evaluation questions.

## Public Support

For non-sensitive public questions, you may use GitHub Issues for:

- broken links
- documentation errors
- unclear public wording
- governance file corrections
- repository metadata or public-readiness feedback

Please do not include security-sensitive information, private implementation details, dashboard logic, detection rules, telemetry assumptions, customer data, credentials, screenshots of private systems, incident records, or enterprise deployment assumptions in public issues.

## Commercial Inquiries

For licensing, enterprise evaluation, design-partner conversations, acquisition inquiries, or Private Enterprise Edition access, contact the repository owner through the public GitHub profile or the contact path referenced in the repository README.

Commercial discussions should not be opened as public issues when they involve pricing, private capabilities, implementation details, buyer requirements, dashboard logic, customer environments, or confidential evaluation criteria.

## Security Reporting

Security issues must follow SECURITY.md.

Do not publish vulnerability details, secrets, tokens, detection logic, telemetry details, private infrastructure references, customer information, incident data, or implementation-sensitive findings in public issues or discussions.

## Public Edition vs. Private Enterprise Edition

Public support covers the Public Edition only. It does not include:

- private deployment support
- production SOC workflow design
- detection rule or telemetry pipeline development
- threat-intelligence integration support
- authentication or access-control implementation support
- customer environment review
- private dashboard logic review
- commercial architecture transfer

Private Enterprise Edition support, if available, requires a separate written agreement.

## Response Expectations

This repository is maintained as a public product and governance surface. Public issue response times are best-effort and depend on severity, relevance, and maintainer availability.

Security reports and commercial inquiries are prioritized through the appropriate private channels.
